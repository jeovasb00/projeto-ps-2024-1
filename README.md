# RABBIT

## Informações Gerais
O RABBIT é um aplicativo inovador que visa tornar o aprendizado de informática uma experiência acessível, envolvente e divertida para crianças e idosos. Integrando elementos de jogos e interatividade, o RABBIT oferece uma abordagem única para ensinar conceitos básicos de informática.

A interface do aplicativo é projetada para ser intuitiva e amigável, permitindo que até mesmo os usuários menos experientes naveguem facilmente. O RABBIT oferece uma variedade de missões educacionais que abrangem desde a navegação na internet até conceitos de segurança cibernética.

Ao progredir no aplicativo, os usuários recebem feedback personalizado, destacando suas conquistas e identificando áreas para melhorias. O RABBIT é adaptável, ajustando-se às necessidades individuais de cada usuário e oferecendo acesso em dispositivos móveis e desktops para aprendizado em qualquer lugar e a qualquer momento.

### Objetivos

#### Objetivo Geral
Capacitar crianças, idosos e quem deseja aprender a se tornarem proficientes em informática, reduzindo a lacuna de habilidades digitais e promovendo a inclusão digital em todas as faixas etárias.

#### Objetivos Específicos
-Aumentar a Proficiência em Informática: O objetivo principal do RABBIT é aumentar a proficiência em informática de crianças e idosos, capacitando-os a compreender e utilizar com confiança as tecnologias digitais;

-Reduzir a Lacuna de Habilidades Digitais: O aplicativo visa diminuir a lacuna de habilidades digitais, proporcionando uma plataforma acessível e interativa para aprender conceitos básicos de informática;

-Oferecer uma Experiência Divertida e Engajante: O aplicativo busca oferecer uma experiência de aprendizado divertida e envolvente, utilizando elementos de jogos e interatividade para tornar o processo de aprendizado mais atraente.

### Situação problema

Em muitas comunidades, especialmente em áreas rurais ou em regiões com acesso limitado à educação em tecnologia, crianças e idosos enfrentam desafios significativos ao tentar adquirir habilidades básicas de informática. As escolas podem não oferecer currículos adequados em tecnologia, e muitos idosos podem não ter acesso a programas de treinamento em informática. Isso pode resultar em uma lacuna de habilidades digitais que os coloca em desvantagem no mundo moderno, onde o uso da tecnologia é cada vez mais importante para a educação, comunicação e oportunidades de trabalho.

### Introdução

No mundo moderno, a capacidade de entender e utilizar a tecnologia digital tornou-se uma habilidade essencial para o sucesso em muitas áreas da vida. No entanto, para muitas crianças e idosos, especialmente aqueles em comunidades com acesso limitado à educação em informática, adquirir essas habilidades pode ser um desafio significativo.

O projeto RABBIT surge como uma resposta a essa lacuna de habilidades digitais, oferecendo uma solução inovadora e acessível para capacitar crianças e idosos a se tornarem proficientes em informática. Este projeto visa não apenas fornecer conhecimento técnico, mas também transformar o processo de aprendizado em uma experiência interativa e divertida, semelhante a um jogo.

# Diagrama de Caso de Uso

## Tela de Login

## Descrição:

- Este diagrama de caso de uso descreve as interações entre o usuário e a tela de login do aplicativo.

## Atores:
- Usuário

## Casos de Uso:

### 1. Realizar Login
   Descrição: O usuário realiza o login no aplicativo utilizando suas credenciais.
   - Ator Principal: Usuário
   - Fluxo Principal:
     1. O usuário abre o aplicativo "Rabbit".
     2. O sistema exibe a tela de login.
     3. O usuário insere seu nome de usuário e senha.
     4. O usuário clica no botão de "Entrar".
     5. O sistema verifica as credenciais do usuário.
     6. O sistema autentica o usuário e o redireciona para a tela principal do aplicativo.

### 2. Recuperar Senha
   Descrição: O usuário solicita a recuperação da senha caso tenha esquecido.
   - Ator Principal: Usuário
   - Fluxo Principal:
     1. O usuário abre o aplicativo "Rabbit".
     2. O sistema exibe a tela de login.
     3. O usuário clica no link "Esqueceu a senha?".
     4. O sistema solicita o endereço de e-mail do usuário.
     5. O usuário insere seu endereço de e-mail e clica no botão "Enviar".
     6. O sistema envia um e-mail com instruções para redefinir a senha.

### 3. Registrar Novo Usuário
   Descrição:O usuário cria uma nova conta no aplicativo.
   - Ator Principal:Usuário
   - Fluxo Principal:
     1. O usuário abre o aplicativo "Rabbit".
     2. O sistema exibe a tela de login.
     3. O usuário clica no link "Registrar".
     4. O sistema redireciona o usuário para a tela de registro.
     5. O usuário preenche o formulário de registro com suas informações pessoais.
     6. O usuário clica no botão "Registrar".
     7. O sistema cria uma nova conta para o usuário e o redireciona para a tela de login.

## Extensões:
- Caso de Uso 1a: Credenciais Inválidas
  - Descrição:Se as credenciais fornecidas pelo usuário forem inválidas, o sistema exibirá uma mensagem de erro.
  - Fluxo Alternativo:
    1. O sistema exibe uma mensagem de erro informando que as credenciais fornecidas são inválidas.

- Caso de Uso 2a: E-mail Inválido
  - Descrição: Se o usuário inserir um endereço de e-mail inválido durante a recuperação da senha, o sistema exibirá uma mensagem de erro.
  - Fluxo Alternativo:
    1. O sistema exibe uma mensagem de erro informando que o endereço de e-mail fornecido é inválido.

- Caso de Uso 3a: Registro Incompleto
  - Descrição: Se o usuário não preencher todos os campos obrigatórios durante o registro, o sistema exibirá uma mensagem de erro.
  - Fluxo Alternativo:
    1. O sistema exibe uma mensagem de erro informando que todos os campos obrigatórios devem ser preenchidos.

# Tela de Lição

## Descrição:
Este diagrama de caso de uso descreve as interações entre o usuário e a tela de lição do aplicativo.

## Atores:
- Usuário

## Casos de Uso:

### 1. Visualizar Lição
   Descrição: O usuário visualiza uma lição específica no aplicativo.
   - Ator Principal: Usuário
   - Fluxo Principal:
     1. O usuário navega até a tela de lições no aplicativo "Rabbit".
     2. O sistema exibe uma lista de lições disponíveis.
     3. O usuário seleciona uma lição específica para visualizar.
     4. O sistema carrega o conteúdo da lição selecionada e o exibe na tela.
     5. O usuário lê o conteúdo da lição e pode interagir com elementos interativos, como vídeos, quizzes, exercícios práticos, etc.
     
### 2. Marcar Lição como Concluída
   Descrição: O usuário marca uma lição como concluída após terminá-la.
   - Ator Principal: Usuário
   - Fluxo Principal:
     1. O usuário completa a visualização da lição conforme descrito no Caso de Uso 1.
     2. O sistema exibe uma opção para marcar a lição como concluída.
     3. O usuário seleciona a opção para marcar a lição como concluída.
     4. O sistema registra a lição como concluída para o usuário.

### 3. Avaliar Lição
   Descrição: O usuário avalia a lição após concluí-la.
   - Ator Principal: Usuário
   - Fluxo Principal:
     1. O usuário completa a visualização da lição conforme descrito no Caso de Uso 1.
     2. O sistema exibe uma opção para avaliar a lição.
     3. O usuário seleciona a opção para avaliar a lição.
     4. O sistema solicita ao usuário que atribua uma classificação (por exemplo, de 1 a 5 estrelas) e forneça feedback adicional, se desejado.
     5. O usuário envia a avaliação.
     6. O sistema registra a avaliação da lição.

## Extensões:
- Caso de Uso 1a: Lição Indisponível
  - Descrição: Se não houver lições disponíveis para visualização, o sistema exibirá uma mensagem informativa.
  - Fluxo Alternativo:
    1. O sistema exibe uma mensagem informando que não há lições disponíveis no momento.

- Caso de Uso 2a: Lição Já Concluída
  - Descrição: Se o usuário tentar marcar uma lição como concluída que já tenha sido marcada anteriormente, o sistema exibirá uma mensagem de aviso.
  - Fluxo Alternativo:
    1. O sistema exibe uma mensagem informando que a lição já foi marcada como concluída.

- Caso de Uso 3a: Avaliação Não Enviada
  - Descrição: Se o usuário decidir não enviar uma avaliação após concluir uma lição, o sistema não registrará a avaliação.
  - Fluxo Alternativo:
    1. O sistema não registra a avaliação da lição.
